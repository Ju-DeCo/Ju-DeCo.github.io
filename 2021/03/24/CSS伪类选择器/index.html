2. UI元素相关的伪类

UI 元素相关的伪类针对 HTML 中的表单元素进行操作。

用于 UI 元素状态的伪类有： :focus 、:enabled 、:disabled、:checked、:read-only、:read-write、:default 和 :indeterminate。另外还有 :hover 和 :active。

用于 UI 元素的有效性验证的伪类有： :invalid、:valid、:required、:optional、:in-range 和 :out-of-range。

2.1 :focus

:focus 选择器被用来指定元素获得焦点时使用的样式，主要在文本框控件获得焦点并进行文本输入时使用，使用时可结合属性选择器，列如：

2.2 :enabled 和 :disabled

:enabled 选择器被用来指定当元素处于可用状态时的样式，:disabled 相反。

2.3 :read-only 和 :read-write

:read-only 选择器被用来指定当元素处于只读状态时的样式，:read-write 相反。

2.4 :checked 和 :indeterminate

这两个选择器主要被应用于单选按钮和复选框元素。:checked 选择器用来指定当表单中的单选按钮或复选框处于选中状态时的样式。

:indeterminate 选择器有如下 3 种应用场景：

  1. 表单中拥有相同 name 属性值的所有单选按钮都未被选中时
  2. indeterminate 属性被 JavaScript 设置为 true 的复选框控件
  3. 处于不确定状态的<progress>元素

2.5 :default

:default 选择器可用在 <button>、<input type=”checkbox”>、<input type=”radio”>以及<option>元素上。允许进行多项选择的分组元素也可以具有多个默认值，该情况下所有默认值都使用 :default 选择器表示。

2.6 :invalid 和 :valid

:invalid 选择器用于在表单元素中输入非法值时设置指定样式。:valid 选择器用来指定当元素内容通过指定的检查，或元素内容符合所规定格式时的样式。这两个选择器只作用于能指定区间值的元素。

2.7 :required 和 :optional

:required 选择器用来指定允许使用且已经指定 required 属性的<input>元素、<select>元素以及<textarea>元素的样式。

:optional 选择器用来指定允许使用但 尚未指定 required 属性的<input>元素、<select>元素以及<textarea>元素的样式。

2.8 :in-range 和 out-of-range

:in-range 选择器用来指定当元素的有效值被限定在一段范围内，且实际的输入值在该范围时的样式。

:out-of-range 选择器用来指定当元素的有效值被限定在一段范围内，但实际的输入值超出该范围时使用的样式。

3. 结构伪类

3.1 :root 和 :empty

:root 将样式绑定到页面的根元素，即<html>元素。

:empty 用来选择没有任何内容的元素，空格也算内容。

3.2 :first-child 和 :last-child

:first-child 选择器用来选择第一个子元素（所有的第一个子元素都会被选择）。:last-child 用来选择元素的最后一个子元素。

3.3 :nth-child(n) 和 :nth-last-child(n)

:nth-child (n)用来定位父元素的第一个或多个特定的子元素。其中 n 是参数，可以是整数、表达式和关键词（odd、even等），参数 n 的起始值始终是1，如果参数 n 的值为 0 ，选择器将选择不到任何匹配的元素。下面选择器将匹配表格中相应的行：

  1. tr:nth-child (2n+1) ：匹配奇数行的 tr
  2. tr:nth-child (2n) ：匹配偶数行的 tr
  3. tr:nth-child (odd) ：匹配奇数行的 tr
  4. tr:nth-child (even) ：匹配偶数行的 tr
  5. tr:nth-child (5) ：匹配第5行的 tr

:nth-last-child (n)选择器用于选择父元素中倒数第 n 个位置的元素或特定元素，与 :nth-child (n) 的区别在于，:nth-chilid (n)选择元素时从上往下数，而 :nth-last-child (n) 从下往上数。

3.4 :first-of-type 和 :last-of-type

:first-of-type 选择器类似于 :first-child 选择器，不同在于前者指定元素的类型，主要用来定位父元素下某个类型的第一个子元素。:last-of-type 选择器和 :first-of-type 选择器的功能一样，不同在于前者选择父元素下某个类型的最后一个子元素。

例5 :nth-child(n) :nth-last-child(n) :first-of-type :last-of-type

3.5 :nth-of-type(n) 和 :nth-last-of-type(n)

:nth-of-type (n) 和 :nth-child (n) 选择器非常类似，区别在于nth-child (n) : 匹配父元素中的第 n 个子元素，元素类型没有限制。 nth-of-type (n) : 匹配同类型中的第n个同级兄弟元素。

:nth-last-of-type (n) 选择器用法和 :nth-of-type(n) 选择器一样。

3.6 :only-child 和 :only-of-type

:only-child 选择器匹配属于父元素的唯一子元素的每个元素。

:only-of-type 表示一个元素有很多子元素，而其中只有一种类型的子元素是唯一的，使用该选择器可以选中这个元素中这种类型的子元素。
