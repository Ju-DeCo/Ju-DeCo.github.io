HTML表单–创建表单

在网页中，用<form>和</form>这对标签来创建表单，<form>元素通常是一些用来采集用户输入信息的表单控件，<form>元素不能嵌套使用，一般<form>标签应该至少带有 action 和 method 两个属性，还可包含以下属性：enctype、novalidate、target、autocomplete 和 accept-charset。

1. action 属性

action 属性指明表单提交后对数据的处理，通常该属性的值是一个地址，也就是表单收集到的信息将要被传递到的地址。列如，对于登录表单，用户输入的登录名和密码信息可能被传送到 Web 服务器上一个以 ASP.NET 编写的页面，叫作 login.aspx。若该页面与当前页面在同一目录下，使用如下：

2. method 属性

浏览器向服务器发送表单数据时可以使用两种方法：HTTP get 和 HTTP post。使用<form>标签的 method 属性指定使用哪一种方法，该属性的两种取值如下：

  • get ：使用 HTTP get 方法向服务器发送表单数据，表单数据被附加在<form>元素中有 action 属性指定的 URL 尾部。表单数据与 URL 之间使用问号分隔，在问号之后是各表单控件的"名称/值"对。"名称/值"对之间使用 & 分隔。
  • post ：使用 HTTP post 方法向服务器发送来自表单的数据，表单数据将在 HTTP 头部中透明的传送

以 HTTP get 方法传递表单数据的一个好处是可以收藏地址，但该方法在发送诸如密码或信用卡细节等敏感信息时，不应该使用 HTTP get 方法。因为这些信息会成为 URL 的一部分，并且对所有人可见。以下情景不应该使用 HTTP get 方法：

(1) 处理敏感信息

(2) 更新数据源

(3) 表单中包含文件上传控件，文件无法在 URL 中传递

在使用 HTTP post 方法向服务器发送来自表单的数据时，表单数据将在 HTTP 头部中透明的传递。

3. id 和 name 属性

id 属性唯一标识页面中的<form>元素。name 属性是 id 属性的前任，与 id 属性一样，取值在文档中应该保持唯一

4. enctype 属性

如果使用 HTTP post 方法向服务器发送数据，则可以使用 enctype 属性指定浏览器在将数据发送到服务器之前对其进行编码，浏览器通常支持以下三种编码方式：

(1) application/x-www-form-urlencoded：这是大多数表单使用的标准方式，浏览器使用该方式是因为一些字符不能直接发到 Web 服务器，这些字符会被其他字符代替

(2) multipart/form-data：允许把数据分成部分再传送，每一个连续部分以其在表单中出现的顺序对应于一个表单控件，通常用于访问者需要向服务器上传文件

(3) text/plain：以纯文本形式不加修改的将数据发送至服务器

如果没使用 enctype 属性，浏览器使用第一个值，一般情况，只有在表单允许用户向服务器上传文件，或用户有可能使用 ASCII 字符时，才需要使用该属性。

5. accept-charset 属性

在创建网站时，开发人员不会将网站设计成能理解所有语言，该属性可以指定一系列用户能够输入，并且之后服务器可以处理的字符编码，该属性的值应该是一个有空格或逗号分隔的字符集列表。如果没设置该属性，那么任何字符集都是有效的。

6. novalidate 属性

该属性是一个布尔属性，用以指定表单在提交时是否进行校验，如果使用了该属性，则关闭对表单内所有元素的有效检验，如希望只取消对表单中少部分内容验证，可以在不需要验证的表单控件上使用该属性。

7. target 属性

该属性指定一个命名窗口或关键字，用于处理表单提交，该属性的取值与 <a> 标签的 target 属性一样。

8. autocomplete 属性

该属性用来指明浏览器是否应该自动填写表单值。默认值为 on ，如果将它设置为 off ，则指明浏览器不应该自动填写任何内容。

9. 多行文本输入控件

在表单中，创建多行文本使用的是<textarea>元素，该元素常用的属性如下：

  • name ：控件的名称，在发送至服务器的“名称/值”对中使用
  • rows ：用于指定应该具有的文本行数
  • cols ：用于指定应该具有的文本列数

默认情况下，文本满一行后会自动换行，但数据提交到服务器是以单行文本的形式。如果用户希望提交到服务器后如他们在屏幕上看到的一样进行换行，可使用 <textarea>的 wrap 属性，该属性有两取值：

  • soft ：默认值，当文本自动换行，数据以单行文本形式传送给服务器，通过回车键换行的，会以换行符形式传送给服务器
  • hard ：无论文本在哪换行，都将以新行的形式传送给服务器，使用该值时，必须设置 cols 属性。
