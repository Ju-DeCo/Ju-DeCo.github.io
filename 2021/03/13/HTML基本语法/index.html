HTML基本语法

1. 标签与元素

HTML 文档中尖括号括起来的字符就是 HTML 的“标签”。且标签大多成对出现，它们中有开标签和闭标签

​ 开标签:<html> 闭标签：</html> 以”/“开头

一对标签及二者之间的内容称作“元素”。

2. 核心元素

  • <body>元素：包含希望在浏览器主窗口中显示的信息。
  • <head>元素：主要包含页面的相关信息（此处不是页面的主题内容）

2.1 关于DOCTYPE

类型声明，将文档类型声明为HTML。

2.2 <html>元素

出现在 DOCTYPE 声明之后，包含以下几个属性：id、dir、lang。

2.3 <head>元素

它是<html>后出现的第一个标签，通常包含一个 <title> 元素，用以指定文档的标题。

​ <head> 元素还可以包含以下元素的任意顺序出现：

  • <base >元素：用以为页面指定基础URL地址。通过这种设置，浏览器会以在相对地址前加上基础地址的方式得到完整的绝对地址。
  • <link>元素：用于链接外部文件，例如CSS样式表等。
  • <style>元素：用于在文档内包含CSS规则。
  • <script>元素：用于在文档内包含脚本。
  • <meta>元素：包含文档的相关信息，比如一段描述或作者姓名等。

2.4 <title>元素

用来用来指定网页的标题，是 <head> 元素的子元素。

<title> 元素应该只能包含标题文本，不可以包含任何其他元素。<title>元素可以包含以下属性：id、dir、lang

2.5 链接与样式表

<link> 元素用来添加样式表，使用 href 属性指向Web上的某个资源。此处的 href 不是为了在单击链接时打开一个新的页面或网址，而是指向为当前页面提供样式信息的文件的位置。使用 rel 属性指明链接的文档是样式表。

在页面中添加脚本使用 <script> 元素，然后后面添加 src 属性，指向需要使用的 JavaScript 文件的位置。

2.6 常见的内容元素

  • 标题的6个级别：<h1><h2><h3><h4><h5>及<h6>
  • 段落<p>、格式化小节<pre>、断行<br/>、块引用<blockquote>、及地址<address>
  • 分组元素：<div>、<header>、<hgroup>、<nav>、<section>、<article>、<footer>、<aside>以及<hr>
  • 呈现性元素：<b>、<i>、<sup>以及<sub>
  • 短语元素：<em>、<strong>、<abbr>、<dfn>、<blockquote>、<q>、<cite>、<code>、<kbd>、<var>以及<samp>
  • 列表：无序列表<ul>、<li>，使用<ol>、<li>的有序列表，以及使用<dl>、<dt>及<dd>定义的列表。
  • 编辑元素：<ins>和<del>

3.1 核心属性

可在大多数HTML中使用的 4 个核心元素是： id、title、 class、和style

(1) id 属性

​ 用来唯一标识页面中的一个元素，或者用来指定一个 CSS 样式或一段 JavaScript 代码只被应用于文档中的该元素。

​ 例如可用 id 属性区分两个段落元素：

id 属性取值需遵守下列规则：

  • 必须以字母(大小写均可)开头。之后可接任意数量的字母、数字、横线(-)、下划线(_)、分号(;)、以及句号(.)。不能以数字、横线、下划线、分号和句号开头。
  • 在文档中必须保持唯一性。同一个 HTML 页面不允许存在两个取值相同的 id 属性，这种情况应该由 class 属性处理。

(2) class 属性

​ 用于指定某元素属于某一特定“类型”。

​ class 属性的取值也可以是一个以空格分隔的 class 名称列表。

(3) title 属性

​ title 属性为元素提供标题。该属性的取决于包含它的元素，经常用作提示标签或在元素载人时显示。

(4) style 属性

​ style 属性用于在元素内部指定CSS规则。

3.2 国际化

(1) dir 属性

​ 用于指定浏览器中文本的显示方向：从左往右 或 从右往左。该元素应该在 <html> 中使用而非 <body> 中。该属性的两个取值：

  • ltr ：从左往右显示（默认）
  • rtl ：从右往左显示

(2) lang 属性

​ 用于指定文档中使用的主要语言。常见取值如下：

  • ar ：阿拉伯语
  • en ：英语
  • en-us ：美国英语
  • zh ：中文

3.3 全局属性

全局属性指可以对任何元素使用的属性。

属性 描述
accesskey 规定激活元素的快捷键
contenteditable 规定元素内容是否可编辑
contextmenu 规定元素的上下文菜单，上下文菜单在用户单击元素时显示
data-* 用于存储页面或应用程序的私有定制数据
draggable 规定元素是否可拖动
designMode 指定整个页面是否可编辑，当页面可编辑时，页面中任何支持 contentEditable属性的元素都变成可编辑状态
dropzone 规定在拖动被拖动数据时是否进行复制、移动或链接
hidden 规定元素仅仅在视觉上看不见，占据的空间位置仍然存在
spellcheck 对用户输入的文本内容进行拼写检查和语法检查
tabindex 规定元素的 Tab 键次序
translate 规定是否应该翻译元素内容

4. 文本格式化

4.1 空格

​ 当一段文本中的两个字之间出现多个连续的空格时，默认情况下屏幕上只有一个空格会被显示，这被称为“空格压缩”。在 HTML 文档中另起一新行，或者当有多个连续的空行时，只做一个空格处理。

​ 如果页面需要显示多个连续的空格，有两种方法：

  • 使用 <pre> 标签
  • 使用空格符号 “&nbsp;” 

除了空格外，还有一些符号需要用代码替代，常用的如下

< &lt; > &gt;
× &times; § &sect;
© &copy; ® &reg;

4.2 使用标题

​ HTML 提供了6种级别的标题 <h1>、<h2>、<h3>、<h4>、<h5> 以及 <h6> 其中最大的字体为 <h1>，这6和标题元素都可以包含以下通用属性： class 、id 、style 、title 、dir 、和 lang

4.3 使用<p>元素创建段落

​ 使用 <p> 可以分段显示网页中的文本，让文章具有段落之分。开标签 <p> 和闭标签 </p> 之间的所有文本都在一个段落内。

4.4 使用<br/>和<hr/>标签

​ 使用 <br/> 元素，可以将段落文本换行显示，不需要开闭标签对，一个 <br/> 代表一次换行。

​ 水平线标签 <hr/> 可在网页中添加一条水平线。

4.5 使用<pre>元素预格式化文本

​ <pre>  </pre>中的元素能够保持编写时的格式，通常用来显示源代码等。

4.6 使用字体样式标签

  • <b> ：用以粗体显示文本
  • <em> ：用以定义强调的内容，通常显示为斜体
  • <strong> ：用以定义语气更强的强调内容
  • <i> ：用以显示斜体效果，和<em>类似
  • <sup> ：实现文本上标效果
  • <sub> ：实现文本下标效果
  • <u> ：用以给文本添加下划线
