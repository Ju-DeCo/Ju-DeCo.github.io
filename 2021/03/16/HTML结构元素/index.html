HTML结构元素

​ 在HTML5对分组元素进行扩展之前，最常用的组容器就是 div 元素。针对文章的不同部分，需要很多的 div 元素，HTML5引入了一些结构性元素改变了这种情况。

元素 描述
section section元素可以用于区域的章节表述
header 页面主体的头部，head元素不可见，而header元素往往包含在body元素中
footer 页面的底部，即页脚
nav 专门用于菜单导航、链接导航的元素
article 用于表示一篇文章的主题内容，一般为文字集中显示的区域

1. section 元素

​ section 元素主要用来对网站或应用程序中页面上的内容进行分块，section 元素通常有标题和内容组成，但 section 元素不是容器元素，所以不能用 CSS 渲染。当一个容器需要直接定义样式或通过脚本控制行为时，应使用 div 元素。

​ 这是 section 元素的典型结构，有标题和内容组成。如果一段没有标题不建议使用 section 元素。

  • 如果一块内容相对比较独立、完整，应使用 article 元素
  • 如果要将一块内容分成几段，应使用 section 元素

2. header 元素

​ header 元素是一种具有引导和导航作用的结构元素，通常用来放置整个页面或页面内的内容区块的标题。注意 head 与 header 元素的不同，head 元素是文档的所有头部元素容器，而 header 元素是 body 元素的一个结构元素。可在 article 中使用 headre 元素，但不能在footer 、 address 或另一个 header 元素中使用 header 元素。

3. footer 元素

​ footer 元素可作为内容块的脚注，比如在父级内容块中添加注释，或者在网页中添加版权信息等。一个网页可重复使用 footer 元素，还可以为 article 元素和 section 元素添加 footer 元素。

4. nav 元素

​ nav 元素用来作为页面导航的链接组，一个页面可以拥有多个 nav 元素，作为页面整体或不同部分的导航，一般来说，nav 元素适用于以下场景：传统导航条、侧边栏导航条、页内导航、翻页操作。可以像使用 <div> 标签、<span> 标签一样使用 <nav> 标签，还可以添加 id 或 class 属性。
